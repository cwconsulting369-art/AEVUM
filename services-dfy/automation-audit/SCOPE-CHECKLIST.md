# Scope-Checklist — Automation Audit

## In-Scope

| Kategorie | Item | Aufwand-Estimate |
|---|---|---|
| **Intake** | Kick-off Call (Ziele, Erwartungen, Logistik) | 1,5h |
| **Intake** | Ausgabe + Auswertung Onboarding-Fragebogen (strukturiert, 40-60 Fragen) | 3h |
| **Discovery** | Stakeholder-Interviews (bis 5 Personen, je 60-90 min) | 6-8h |
| **Discovery** | Tool-Stack-Inventarisierung (alle aktiven Tools, Lizenzen, Kosten) | 3-4h |
| **Discovery** | Prozess-Dokumentation Ist-Stand (bis 6 Kern-Workflows, Flowchart-Level) | 6-10h |
| **Analyse** | Ineffizienz-Scoring pro Workflow (Zeit/Fehler/Risiko-Matrix) | 4-6h |
| **Analyse** | Automation-Kandidaten-Identifikation + Effort/Impact-Einschätzung | 4-5h |
| **Analyse** | ROI-Kalkulation (konservative Schätzwerte, keine Garantien) | 3-4h |
| **Analyse** | Tool-Stack-Bewertung (Behalten / Ersetzen / Ergänzen) | 2-3h |
| **Roadmap** | Quick-Win-Sheet (Top 5 Automationen, sofort umsetzbar) | 2h |
| **Roadmap** | 90-Tage-Phasenplan (Projekte, Sequenz, Aufwände, Abhängigkeiten) | 4-5h |
| **Deliverable** | Findings-Deck (Executive Summary + Detail-Anhang, max. 30 Slides) | 4-6h |
| **Deliverable** | Readout-Präsentation (bis 2 Sessions à 90 min) | 3h |
| **Deliverable** | Priorisierungs-Workshop (live, 60 min, mit Entscheider) | 1h |
| **Handover** | Vollständiges Handover-Package (alle Docs + Video-Walkthrough) | 2h |

**Gesamt-Aufwand-Estimate Standard-Audit: 48-65h intern**

---

## Out-of-Scope

| Was Customer NICHT bekommt | Begründung |
|---|---|
| Umsetzung / Bau von Automationen | Das ist ein separates Implementation-Projekt |
| Technische Tool-Migrationen oder Datentransfers | Requires separate Scoping |
| DSGVO-/Compliance-/Security-Bewertung | Erfordert Rechtsexperten, nicht Automation-Strategen |
| HR-Prozess-Optimierung (Hiring, Payroll, Org-Design) | Out of AEVUM's Kompetenzbereich |
| Code-Review bestehender Custom-Lösungen | Nur funktionale Bewertung, kein technisches Audit |
| Garantierte ROI-Zahlen | Schätzwerte basierend auf Industrie-Benchmarks, keine Testierung |
| Laufendes Monitoring nach Handover | Endet mit Übergabe, Retainer ist separates Produkt |
| Training des Customer-Teams auf neue Tools | Nur Empfehlung, kein Live-Training |
| Vendor-Verhandlungen oder Tool-Purchasing | Liegt beim Customer |
| Mehr als 6 Kern-Workflows (Standard-Tier) | Extended/Enterprise für mehr Tiefe |

## Voraussetzungen Customer-Side

| Kategorie | Anforderung | Kritikalität |
|---|---|---|
| **Access** | View-Rechte auf alle relevanten Tools | Pflicht |
| **Access** | Admin-Rechte für Tool-Stack-Details (optional aber hilfreich) | Nice-to-have |
| **Daten** | SaaS-Kostenübersicht / Tool-Rechnungen letzter 3 Mo. | Pflicht |
| **Daten** | Grobe Zeiterfassungsdaten oder Schätzungen (wie lang dauern Prozesse) | Pflicht |
| **Zeit** | Kick-off Call: 90 min (Entscheider + 1-2 Ops-Verantwortliche) | Pflicht |
| **Zeit** | Stakeholder-Interviews: 5 x 60-90 min (verschiedene Rollen) | Pflicht |
| **Zeit** | Readout-Session: 90 min (Entscheider anwesend) | Pflicht |
| **Team** | Klarer Ansprechpartner (Single Point of Contact) auf Customer-Seite | Pflicht |
| **Mindset** | Bereitschaft zur offenen Darstellung von Problemen und Fehlern | Pflicht |
| **Reifegrad** | Mindestens 6 Monate Betrieb aktueller Prozesse | Pflicht |

## Quality-Standards

AEVUM sagt "Done" wenn folgende Kriterien erfüllt sind:

| Standard | Kriterium |
|---|---|
| **Vollständigkeit** | Alle in Scope definierten Deliverables sind übergeben |
| **Prozess-Dokumentation** | Jeder dokumentierte Workflow enthält: Trigger, Schritte, beteiligte Tools, Handoff-Punkte, bekannte Fehlerquellen |
| **Ineffizienz-Matrix** | Jede Lücke hat: Zeit-Verlust-Schätzung, Fehlerrate-Einschätzung, Automatisierbarkeits-Score (1-5) |
| **Roadmap** | Enthält: Phasen, Projekte pro Phase, Abhängigkeiten, Effort-Estimate, Priority-Score |
| **ROI-Kalkulation** | Mindestens 3 Szenarien (konservativ / realistisch / optimistisch) mit Quellenangabe der Annahmen |
| **Findings-Deck** | Executive Summary max. 5 Slides, vollständiger Detail-Anhang, keine generischen Empfehlungen ohne Datenbasis |
| **Quick-Win-Sheet** | Top 5 Automationen mit: Tool-Empfehlung, geschätzter Aufwand, geschätzter Nutzen, nächste konkrete Schritte |
| **Readout** | Customer hat aktiv Feedback gegeben, offene Fragen sind dokumentiert und beantwortet |
| **Handover-Package** | Alle Dokumente in strukturiertem Ordner, Video-Walkthrough vorhanden (min. 20 min) |
| **Customer Sign-Off** | Schriftliche Bestätigung des Customers dass Findings vollständig und verständlich sind |

## Change-Request-Policy

| Szenario | Handling |
|---|---|
| Customer möchte zusätzliche Workflows in den Audit einbeziehen (über Scope hinaus) | Change-Request-Formular, Kosten-Schätzung innerhalb 48h, Customer-Approval vor Start |
| Customer möchte tiefere technische Analyse einzelner Tools | Bewertet als Add-On (€800-2.500 je nach Aufwand), separates SOW |
| Customer möchte nach Audit-Abschluss Roadmap-Revision | 1x kostenlose Minor-Revision innerhalb 14 Tage nach Handover; danach €150/h |
| Customer stellt nachträglich fest, dass kritische Prozesse nicht erfasst wurden (Customer-Fehler) | Zusatz-Interview möglich, Aufwand wird berechnet (€150/h) |
| AEVUM hat relevante Workflows nicht erfasst (AEVUM-Fehler) | Kostenlose Nachbesserung innerhalb Projekt-Scope |
| Customer-Ansprechpartner wechselt mid-project | Re-Onboarding-Aufwand (max. 3h) wird kostenpflichtig wenn >1 Wechsel |
